# 7주차 학습 내용 요약 (NumPy & 데이터 분석 기초)

## 1. NumPy Axis (축) 이해하기
NumPy의 `axis` 인자는 연산이 수행되는 방향을 결정하며, 해당 축을 따라 요소를 합치고 차원을 축소(collapse)합니다.

### 2차원 배열 (Shape: Row, Column)
- **axis=0 (Row 방향):** 위에서 아래로(열별로) 합계/평균 등을 구합니다. 결과의 Shape는 `(Columns,)`가 됩니다.
- **axis=1 (Column 방향):** 왼쪽에서 오른쪽으로(행별로) 합계/평균 등을 구합니다. 결과의 Shape는 `(Rows,)`가 됩니다.

### 3차원 배열 (Shape: Depth, Row, Column)
- **axis=0 (Depth):** 여러 개의 2차원 레이어를 겹쳐서 같은 위치의 요소끼리 연산합니다. 결과 Shape: `(Row, Column)`
- **axis=1 (Row):** 각 레이어 내부에서 행 방향으로 연산합니다. 결과 Shape: `(Depth, Column)`
- **axis=2 (Column):** 각 레이어 내부에서 열 방향으로 연산합니다. 결과 Shape: `(Depth, Row)`

## 2. NumPy 인덱싱 & 슬라이싱 (Indexing & Slicing)
- **정수 인덱싱:** 차원이 사라짐 (2D -> 1D)
- **슬라이싱:** 차원이 유지됨 (2D -> 2D)
- **linspace:** 결과를 항상 **실수(float)** 타입으로 반환함. (출력 결과에 `.`이 붙는 이유)

---

## 3. NumPy 브로드캐스팅 (Broadcasting)
두 배열의 형상(Shape)이 다르더라도 특정 규칙을 만족하면 연산이 가능하도록 차원을 확장하는 기능입니다.

### 3대 규칙
1. **차원 수 맞추기:** 두 배열의 차원 수가 다르면 더 작은 차원의 배열 앞(왼쪽)에 1을 추가하여 차원을 맞춥니다.
2. **크기 확장:** 특정 차원의 크기가 1인 배열은 다른 배열의 해당 차원 크기에 맞춰 복사(Stretch)됩니다.
3. **오류 발생:** 어느 차원에서도 크기가 1이 아니면서 서로 다르면 브로드캐스팅이 불가능합니다. (예: 2x3과 2x2 연산 불가)

## 4. 벡터 내적 및 행렬 곱셈 (`np.dot`)
단순 요소별 곱셈(`*`)과 달리, 행렬의 수학적 곱셈을 수행합니다.

### 차원 일치 조건 (중요!)
- **조건:** 앞 행렬의 **열(Column)** 개수와 뒤 행렬의 **행(Row)** 개수가 반드시 같아야 합니다.
- **결과 Shape:** (앞 행렬의 행, 뒤 행렬의 열)
- **예시:** `(2, 3) dot (3, 4) -> (2, 4)`

---

## 5. Python 파일 입출력 및 Pandas 기초

### 파일 입출력 커서 관리
- **`r+` 모드:** 읽기와 쓰기가 모두 가능하며, 파일 포인터(커서)는 처음에 위치함.
- **`f.read()`:** 파일의 전체 내용을 읽으며 커서를 **맨 끝**으로 옮김.
- **`f.seek(0)`:** 커서를 파일의 맨 처음으로 강제 이동시킴.
- **`f.tell()`:** 현재 커서의 위치를 반환함.

### Pandas 데이터 확인
- **`read_csv()`:** 첫 번째 줄을 자동으로 헤더(열 이름)로 처리함.
- **`head(n)`:** 헤더를 제외한 실제 데이터의 상위 n줄을 보여줌. (데이터가 n줄보다 적으면 전체를 보여줌)
- **`shape`:** 데이터의 (행, 열) 크기를 반환함.

## 💡 오답 노트 및 꿀팁
- **타입 구분:** 출력값에 `.`이 붙어있다면 (`0.`, `5.`) 이는 정수가 아닌 **실수(float)**임을 의미함.
- **헤더 구분:** Pandas에서 헤더는 데이터 개수(`len`)에 포함되지 않음.
- **행렬 곱셈:** `arr1 * arr2`는 요소별 곱셈(Element-wise)이고, 행렬 곱은 `arr1 @ arr2` 또는 `np.dot(arr1, arr2)`를 사용합니다.
